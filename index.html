    How to export from Alles1 (MS Access)

    1. Open the Alles1 database in MS Access
    2. Click Erstellen → SQL-Abfrage
    3. Paste the following query into the editor:
    SELECT CIPBericht_EEG.Datum, CIPBericht_EEG.AA, CIPBericht_EEG.OA
FROM CIPBericht_EEG
WHERE (((CIPBericht_EEG.Datum)>Now()-(2*365)));
    1. Save the query (Ctrl+S)
    2. Click Externe Daten → Excel and save the file
    3. Upload the saved .xlsx file above

        About · EEG Dashboard

        What is measured

        Each row in the export represents one EEG report session. A doctor is credited with a report whenever their name appears in either the AA or OA column for that date. Role classification (assistant vs. consultant) is based on job-title keywords in the cell, not on column position.

        Who is shown

        Only doctors who meet both activity thresholds are listed:

        • > 35 reports total in the loaded dataset
        • > 20 reports in the last 6 calendar months

        Both assistant doctors (Assistenzarzt/ärztin) and rotation consultants (Oberarzt/in, Facharzt/in, …) are included.

        Monthly rate & projection

        The monthly rate is the total divided by the number of active months — months with more than 5 reports. This prevents a rotation break from artificially lowering the rate. The current calendar month is always counted as active (even if incomplete), so a recently returned doctor is not penalised.

        Projection = rate × rotation-period months. Adjust the period (6 / 12 / custom) and the target (default 800) in the individual-doctor panel.

        Status colours

        • Green — On track: projected ≥ target
        • Amber — Behind: projected ≥ 80 % of target
        • Red — At risk: projected < 80 % of target

        Limitations

        • Role classification relies on keyword matching — unusual job titles may be mis-categorised.
        • The projection assumes a constant future rate equal to past performance.
        • Doctors with fewer than 35 total reports will not appear until they cross the threshold.
        • Data is only as current as the last uploaded export.

        Loading new data

        Click the dataset switcher (database icon, top right on the results screen) → Upload new file… to add a fresh export. Old snapshots can be renamed, annotated, or deleted from the same dropdown.

        Using on multiple browsers or devices

        Browser storage is local and isolated — data saved in Safari is not visible in Chrome, and private/incognito windows always start empty. To carry your data across browsers or devices, use Download snapshot in the dataset switcher to save a .json file, then drag it onto the upload zone in any browser to restore it instantly (no re-parsing of the Excel file needed).
